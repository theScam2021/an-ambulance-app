Key Features by File
====================

Backend Core:
- server/index.js              → Express server, Socket.io setup, MongoDB connection
- server/seed.js               → Creates 3 hospitals, 5 users with default credentials

Models:
- server/models/User.js        → Authentication, password hashing, role-based access
- server/models/Case.js        → Emergency cases with severity, status, assignments
- server/models/Hospital.js    → Hospital resources (ICU, OT, ER), active cases
- server/models/DecisionLog.js → Transparent decision logging with timestamps

Business Logic:
- server/utils/severityCalculator.js  → Rule-based severity scoring (Critical/High/Moderate/Stable)
- server/utils/hospitalSelector.js    → Optimal hospital selection based on resources & load

API Routes:
- server/routes/auth.js        → POST /api/auth/login, /api/auth/register
- server/routes/cases.js       → POST /api/cases, GET /api/cases, PUT /api/cases/:id/status
- server/routes/hospitals.js   → GET /api/hospitals, POST /api/hospitals, PUT /api/hospitals/:id
- server/routes/dashboard.js   → GET /api/dashboard/hospital/:id, /api/dashboard/admin

Frontend Pages:
- client/src/pages/Login.jsx              → Authentication interface
- client/src/pages/AmbulanceDashboard.jsx → Case creation form, severity display
- client/src/pages/HospitalDashboard.jsx  → Live queue, status updates, real-time sync
- client/src/pages/AdminDashboard.jsx     → System metrics, hospital loads, decision logs

Frontend Utilities:
- client/src/utils/api.js      → Axios instance, API methods, JWT token handling
- client/src/utils/socket.js   → Socket.io connection management

Technology Stack
================

Backend:
- Runtime: Node.js
- Framework: Express.js
- Database: MongoDB + Mongoose
- Real-Time: Socket.io
- Authentication: JWT + bcrypt
- Environment: dotenv

Frontend:
- Framework: React 18
- Build Tool: Vite
- Styling: Tailwind CSS
- Routing: React Router v6
- HTTP Client: Axios
- Real-Time: Socket.io Client

Development:
- Process Manager: PM2 (production)
- Concurrency: concurrently (development)
- Hot Reload: Vite HMR + Nodemon

Database Collections
====================

1. users
   - username, password (hashed), role, hospitalId

2. cases
   - patientName, age, vitals, injuryType
   - severityScore, severityCategory
   - assignedHospital, status, createdBy, eta

3. hospitals
   - name, location
   - resources (icuBeds, otAvailable, erCapacity)
   - activeCases

4. decisionlogs
   - caseId, actionType, reason, metadata

Installation Commands
=====================

1. npm install                          # Install backend deps
2. cd client && npm install && cd ..    # Install frontend deps
3. cp .env.example .env                 # Setup environment
4. node server/seed.js                  # Seed database
5. npm run dev                          # Start application
